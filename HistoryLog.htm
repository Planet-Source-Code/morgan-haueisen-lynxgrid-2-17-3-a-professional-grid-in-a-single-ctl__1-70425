Notes:

  1. Default Font size, name, etc. is set using the Font property of the control.  All of the cell's properties are set when a row is added, this includes the default Font.  If you need to change the Font of the entire grid after the cells have been populated with data, then used the public sub FormatCells.
  2. Unicode: This control has partial Unicode Support (Editable Mode does not support Unicode Data Entry because it uses a VB Text Box).
  3. Data Control: Having any Grid control (commercial or not) manage the connection to a database (using a datacontrol object) is a bad idea; especially if the connection is to an MS Access database.

a.       This would force the database to be open the entire time the grid is loaded.  Any crash or power outage could leave the database open and possibly corrupted.

b.       You loose control over a users interaction with the database.

c.       The column order would match the DB table fields order, which is seldom the prefered display.

4.       Grid printing is handled by Exporting the grid data to a csv file and using a third party software application (like Excel) to print.  I started to add printing capibilities to the control but the code to produce a deacent print was larger then the control itself.  There are too many variables to handle (column and text widths, font sizes, paper sizes, how to handle printing beyond the paper’s margens, etc.)  See Demo_Printing for a solution that uses an external Form and Class.

5.       Row grouping is just the ability to hide rows.  How it works is handled outside of the control (see the demos); therefore you can not allow column sorting and expect the “grouped” rows to stay together.

6.       Known Problem: The problem with the sub-classing on non-English versions of Windows is Solved! 
There is one exception; setting the FocusRectHide (property that determines whether the Focus Rectangle is visible when the grid losses the focus), for regional settings where English is not the national language, has a large impact on response time.  Some regional settings cause GotFocus/LostFocus to trigger many times while the grid is redrawn.  This causes the grid to redraw for each Lost and Got focus.  I have eliminated the grid from being locked into a continuous loop, but I can’t find a way to prevent the multiple redraws.  See sub zWndProc1/Case WM_SETFOCUS and WM_KILLFOCUS for more information.

7.       I stopped marking changes to the code and removed all markings and dead/changed code that was commented out..  There are to many changes and the markings were interfering with the code readability. If you want to know what code was changed then you will need to compare this version with version 1.89. 

 

Changes:

1.             Changed Row sizing functionality:

2.             Removed AutoResizeRows and replaced it with AllowWordWrap (giving the ability to turn word wrapping on/off).

3.             Removed RowHeightMax and replaced it with MaxLineCount.  This will limit the number of lines that will be displayed when a cell is word wrapped (0=no limit).

4.             Set the minimum System Row height from the Font text height (set in CreateRenderData).  This way the text will always fit within the row's height.

5.             Changed how a cell’s text is drawn when word wrapping is enabled.  If word wrapping is enabled, and the column is set to word wrap, but the text’s width is less then the column width, then the text is drawn normally (with alignment and ellipses).

6.             Moved the static variables to model level variables. Static variables are very memory intensive and slow (use about 3 times the memory).
